## ⚠️ 注意事项

### 必须做到

1. **仓库必须公开**
   - 评审老师需要能够访问
   - 检查仓库设置 → Visibility → Public

2. **代码在 main 分支**
   - 确保稳定可运行的代码在 main 分支
   - 不要使用 dev 或其他分支

3. **README.md 完整**
   - 这是评审的主要依据
   - 必须包含项目说明、功能特性、使用指南

4. **Release 必须创建**
   - 标签：v1.0.0 或 submission-final
   - 说明完整
   - 地址正确

5. **邮件格式正确**
   - 主题格式：【码上AI】技术挑战赛-团队名称-学校
   - 包含所有必需信息
   - 地址链接正确

### 避免错误

1. ❌ 仓库设置为私有（Private）
2. ❌ 提交了敏感信息（API Key、密码）
3. ❌ README.md 不完整或格式错误
4. ❌ 没有创建 Release
5. ❌ 邮件主题格式错误
6. ❌ 链接地址错误
7. ❌ 代码无法编译
8. ❌ 缺少必要文档

## 📞 遇到问题？

### Git 推送失败

**问题**：`git push` 失败

**解决**：
1. 检查网络连接
2. 使用 Personal Access Token 代替密码
3. 确认仓库已创建
4. 检查远程地址是否正确

### 仓库无法访问

**问题**：评审老师无法访问仓库

**解决**：
1. 确认仓库设置为 Public
2. 检查仓库地址是否正确
3. 尝试在无痕模式下访问

### Release 创建失败

**问题**：无法创建 Release

**解决**：
1. 确认代码已推送到 GitHub
2. 检查标签格式是否正确
3. 确认有仓库的写权限

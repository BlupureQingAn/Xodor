# 刷题系统使用指南

## 功能概述

全新的**刷题系统**提供了完整的题目管理、进度跟踪和统计分析功能，让你的刷题过程更加系统化和高效。

## 主要功能

### 1. 📊 进度跟踪

系统自动记录每道题目的完成情况：

- **❌ 未开始** - 还没有尝试过的题目
- **⏳ 进行中** - 已经尝试但未完全通过
- **✅ 已完成** - 测试用例全部通过
- **⭐ 已掌握** - 多次正确（正确率≥75%且正确次数≥3）

### 2. 🔍 多维度筛选

#### 难度筛选
- 简单
- 中等
- 困难
- 全部难度

#### 题型筛选
- 自动识别题目标签（数组、字符串、动态规划等）
- 支持按题型分类查看

#### 状态筛选
- 未开始
- 进行中
- 已完成
- 已掌握
- 全部状态

#### 搜索功能
- 支持按题目名称搜索
- 实时过滤显示结果

### 3. 📈 统计信息

实时显示刷题进度：

- **总题数** - 题库中的题目总数
- **已完成** - 至少通过一次的题目数
- **已掌握** - 多次正确的题目数
- **总正确率** - 所有尝试的总体正确率
- **进度百分比** - 完成进度的可视化显示

### 4. 📋 详细信息

题目列表显示：

| 列 | 说明 |
|---|---|
| 状态 | 题目完成状态图标 |
| 题号 | 题目序号 |
| 题目 | 题目标题 |
| 难度 | 简单/中等/困难 |
| 题型 | 题目标签（如：数组、字符串） |
| 正确率 | 该题的个人正确率 |
| 尝试次数 | 提交测试的次数 |

### 5. 🎯 快速开始

双击任意题目即可：
- 自动加载题目到主界面
- 清空代码编辑器准备答题
- 开始新的练习

## 使用方法

### 打开刷题系统

1. **菜单方式**：工具 → 刷题系统
2. **快捷键**：`Ctrl+P`

### 筛选题目

1. 使用顶部的下拉框选择筛选条件
2. 在搜索框输入关键词
3. 题目列表会实时更新

### 开始练习

1. 在列表中找到想做的题目
2. 双击题目行
3. 主界面会自动加载该题目
4. 开始编写代码

### 查看进度

- 顶部显示总体统计信息
- 每道题显示个人完成情况
- 通过状态图标快速识别

### 重置进度

如果需要重新开始：
1. 点击"🗑️ 重置进度"按钮
2. 确认操作
3. 所有进度数据将被清空

## 进度自动更新

系统会在以下情况自动更新进度：

### 运行测试后
- 记录尝试次数
- 更新正确次数
- 计算正确率
- 自动更新状态
- 保存最后提交的代码

### 状态自动判定
- 首次尝试 → 进行中
- 全部通过 → 已完成
- 连续3次正确且正确率≥75% → 已掌握

### 错题本联动
- 测试失败自动记录到错题本
- 包含失败原因和代码
- 方便后续复习

## 数据持久化

所有进度数据自动保存到本地：
- 位置：`AppData/Local/CodePracticeSystem/question_progress.json`
- 自动保存，无需手动操作
- 关闭程序后数据不会丢失

## 使用技巧

### 1. 系统化刷题
- 先按难度从简单到困难
- 再按题型集中突破
- 最后查看未完成的题目

### 2. 巩固练习
- 筛选"已完成"但未"已掌握"的题目
- 重复练习提高熟练度
- 争取达到"已掌握"状态

### 3. 查漏补缺
- 筛选"未开始"的题目
- 确保没有遗漏
- 全面覆盖知识点

### 4. 弱项强化
- 查看正确率较低的题目
- 针对性练习
- 提高整体水平

### 5. 题型专项
- 按题型筛选
- 集中练习同类题目
- 掌握解题模式

## 与其他功能的配合

### 错题本
- 自动记录失败的题目
- 可以从错题本快速跳转
- 方便针对性复习

- 包括描述、测试用例、参考答案
- 帮助理解题意

### AI分析
- 测试失败后可以请求AI分析
- 获取错误原因和改进建议
- 加速学习过程

## 注意事项

1. **进度数据**
   - 进度数据基于题目ID
   - 重新导入题库不会丢失进度
   - 但题目ID变化会导致进度无法匹配

2. **正确率计算**
   - 基于所有尝试次数
   - 包括失败的尝试
   - 反映真实水平

3. **状态判定**
   - 自动判定，无需手动设置
   - 基于测试结果自动更新
   - 可以通过重复练习提升状态

4. **数据备份**
   - 建议定期备份进度文件
   - 位于 AppData 目录
   - 可以手动复制保存

## 常见问题

### Q: 如何查看某道题的历史提交？
A: 可以通过"历史记录"功能查看所有提交记录。

### Q: 进度数据会丢失吗？
A: 不会，数据自动保存到本地文件，除非手动删除或重置。

### Q: 可以导出进度数据吗？
A: 进度数据保存在 JSON 文件中，可以手动复制备份。

### Q: 如何提高某道题的状态？
A: 多次正确完成该题，系统会自动提升状态到"已掌握"。

### Q: 重置进度后可以恢复吗？
A: 不能，重置是永久性的，建议操作前备份数据文件。

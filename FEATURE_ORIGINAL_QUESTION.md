# 查看原题功能 - 使用指南

## 📖 功能概述

"查看原题"功能允许用户查看导入题库中的完整题目信息，包括题目描述、测试用例和参考答案。这对于学习和理解题目非常有帮助。

---

## 🎯 使用方法

### 方式一：菜单栏
```
题目 → 查看原题 (Ctrl+Shift+V)
```

### 方式二：工具栏
```
点击工具栏中的 📖 查看原题 按钮
```

### 方式三：快捷键
```
按 Ctrl+Shift+V
```

---

## 📋 对话框内容

### 1. 题目描述标签页 📝

**显示内容**：
- 题目标题（大字体、粗体）
- 难度标签（颜色标识）
  - 🟢 简单 - 绿色
  - 🟡 中等 - 橙色
  - 🔴 困难 - 红色
- 完整的题目描述
- 题目标签（如果有）

**用途**：
- 理解题目要求
- 查看题目分类
- 了解题目难度

---

### 2. 测试用例标签页 🧪

**显示内容**：
- 所有测试用例列表
- 每个测试用例包含：
  - 测试用例编号
  - 输入数据
  - 期望输出

**格式**：
```
测试用例 1
输入：
[输入数据]

期望输出：
[期望输出]
```

**用途**：
- 了解所有测试场景
- 理解输入输出格式
- 设计测试策略
- 调试代码

---

### 3. 参考答案标签页 💡

**显示内容**：
- 完整的参考代码实现
- 代码高亮显示
- 提示信息

**提示**：
> 💡 参考答案仅供参考，鼓励你先独立思考和实现。

**用途**：
- 学习解题思路
- 对比自己的实现
- 理解最佳实践
- 学习代码风格

**建议使用流程**：
1. 先自己尝试解题
2. 遇到困难时查看参考答案
3. 理解思路后自己重新实现
4. 对比优化自己的代码

---

## 🎮 操作按钮

### 开始练习按钮
**功能**：
- 清空当前代码编辑器
- 插入基础代码模板
- 关闭对话框
- 开始新的练习

**使用场景**：
- 看完参考答案后重新实现
- 清空错误代码重新开始
- 按照参考思路自己编写

### 关闭按钮
**功能**：
- 关闭对话框
- 保持当前代码不变

## 💡 使用技巧

### 1. 学习模式
```
流程：
1. 先不看答案，自己尝试
2. 实现后运行测试
3. 如果失败，查看测试用例
4. 理解测试要求后再次尝试
5. 仍然困难时查看参考答案
6. 理解思路后点击"开始练习"
7. 自己重新实现一遍
```

### 2. 对比模式
```
流程：
1. 完成自己的实现
2. 查看参考答案
3. 对比两种实现的差异
4. 学习更优的方法
5. 优化自己的代码
```

### 3. 快速学习模式
```
流程：
1. 直接查看参考答案
2. 理解解题思路
3. 点击"开始练习"
4. 根据理解自己实现
5. 不要直接复制代码
```

## 📊 适用场景

### 场景 1：初学者学习
```
问题：不知道如何开始
解决：
1. 查看题目描述理解要求
2. 查看测试用例了解格式
3. 查看参考答案学习思路
4. 开始练习自己实现
```

### 场景 2：调试困难
```
问题：代码总是测试失败
解决：
1. 查看测试用例找出失败原因
2. 对比输入输出格式
3. 检查边界条件
4. 参考答案的处理方式
```

### 场景 3：学习优化
```
问题：代码能通过但不够优雅
解决：
1. 查看参考答案
2. 对比时间空间复杂度
3. 学习更好的算法
4. 优化自己的实现
```

### 场景 4：复习巩固
```
问题：想复习之前的题目
解决：
1. 查看原题回忆题目
2. 查看测试用例
3. 不看答案自己实现
4. 对比参考答案
```

- ✅ 学习代码风格和技巧
- ✅ 对比优化自己的代码

- 不是所有题目都有测试用例
- 可以根据题目自己设计
- 测试用例帮助理解题意

# 代码刷题系统 - 快速开始指南

## 🚀 5分钟上手

### 1. 启动程序
双击 `build\CodePracticeSystem.exe` 启动程序。

**首次启动**：
- ✅ 自动检测系统中的 C++ 编译器
- ✅ 显示检测结果和版本信息
- ✅ 如果未找到编译器，会提示安装

### 2. 导入题库

#### 方式一：使用示例题库（推荐）
1. 点击工具栏 **📚 导入题库** 或按 `Ctrl+I`
2. 选择 `data/sample_questions` 文件夹
3. 点击确定，等待导入完成
4. 系统会显示导入的题目数量

#### 方式二：导入自己的题库
1. 准备 Markdown 格式的题目文件
2. 点击 **📚 导入题库**
3. 选择题库文件夹
4. 系统自动解析并导入

### 3. 开始刷题

#### 浏览题目
- 使用 **上一题** / **下一题** 按钮切换
- 或在左侧题目列表中点击选择
- 或使用搜索框查找题目

#### 编写代码
1. 在中间的代码编辑器中编写 C++ 代码
2. 代码会自动保存（每 300ms）
3. 可以使用代码模板快速开始：
   - 菜单：**代码 → 插入模板**
   - 选择合适的模板

#### 运行测试
1. 点击 **▶ 运行测试** 按钮或按 `F5`
2. 系统会自动：
   - 编译你的代码
   - 运行所有测试用例
   - 显示测试结果
3. 查看右侧的测试结果：
   - ✓ 绿色表示通过
   - ✗ 红色表示失败
   - 显示通过率和详细信息

#### AI 代码分析
1. 点击 **🤖 AI分析代码** 按钮或按 `F6`
2. 等待 AI 分析完成
3. 查看分析结果：
   - 代码思路
   - 优化建议
   - 涉及知识点

### 4. 查看原题

如果想查看题目的完整信息（包括参考答案）：

1. 点击工具栏 **📖 查看原题** 或按 `Ctrl+Shift+V`
2. 查看三个标签页：
   - **📝 题目描述**：完整的题目说明
   - **🧪 测试用例**：所有测试用例
   - **💡 参考答案**：参考代码实现
3. 点击 **开始练习** 清空代码重新开始
4. 或点击 **关闭** 返回

💡 **提示**：建议先独立思考，实在没思路再查看参考答案。

### 5. 查看错题本

1. 点击菜单 **工具 → 错题本** 或按 `Ctrl+W`
2. 查看所有做错的题目
3. 可以筛选：
   - **全部**：显示所有错题
   - **未解决**：只显示未解决的
4. 双击题目可以快速跳转

### 6. 查看历史记录

1. 点击工具栏 **📊 历史记录** 或按 `Ctrl+H`
2. 查看所有做过的题目
3. 显示信息：
   - 题目 ID
   - 最后修改时间
   - 代码行数

## 🎯 使用技巧

### 1. 代码模板
使用代码模板可以快速开始：
- **基础模板**：最简单的 main 函数
- **输入输出模板**：处理标准输入输出
- **数组处理模板**：处理数组问题
- **字符串处理模板**：处理字符串问题
- **算法模板**：包含常用头文件

### 2. 题目搜索
在左侧题目列表中：
- 输入题目标题关键词搜索
- 使用难度筛选器过滤
- 点击题目快速跳转

### 3. 自动保存
- 代码会自动保存，无需手动保存
- 下次打开同一题目会自动加载
- 保存位置：`data/user_answers/`

### 4. 刷新题库
如果题库文件有更新：
- 按 `F5` 刷新当前题库
- 或使用 `Ctrl+R` 重新导入

### 5. 错题复习
定期查看错题本：
- 重点复习未解决的题目
- 标记已解决的题目
- 双击快速跳转到题目

## ⚙️ 配置说明

### 打开设置
点击菜单 **工具 → 设置** 或按 `Ctrl+,`

### 编译器设置
1. **自动检测**：点击"自动检测"按钮
2. **手动选择**：点击"浏览"选择编译器
3. **测试**：点击"测试"验证编译器

### AI 设置
1. **Ollama 本地模型**：
   - 确保 Ollama 服务正在运行
   - 配置服务地址（默认：http://localhost:11434）
   - 配置模型名称（默认：qwen）

2. **云端 API**（可选）：
   - 输入 API Key
   - 用于备用或更强大的分析

## 📝 题库格式

### Markdown 格式示例

```markdown
# 题目标题

难度：简单

## 题目描述

这里是题目的详细描述...

## 示例

### 示例 1
输入: 1 2 3
输出: 6

### 示例 2
输入: hello
输出: olleh

## 提示

- 提示 1
- 提示 2
```

### 支持的格式
- 标准格式：`输入: xxx` / `输出: yyy`
- 英文格式：`Input: xxx` / `Output: yyy`
- 代码块格式：用 \`\`\` 包裹

## ❓ 常见问题

### Q: 编译失败怎么办？
**A:** 
1. 检查编译器是否正确配置
2. 查看错误提示中的建议
3. 确保代码语法正确

### Q: AI 分析不工作？
**A:**
1. 确保 Ollama 服务正在运行（`ollama serve`）
2. 检查设置中的服务地址
3. 尝试重启 Ollama 服务

### Q: 题目没有测试用例？
**A:**
1. 检查 Markdown 文件格式
2. 确保使用正确的格式标记
3. 可以手动添加测试用例

### Q: 如何添加自己的题目？
**A:**
1. 按照 Markdown 格式编写题目
2. 放在一个文件夹中
3. 使用导入功能导入
